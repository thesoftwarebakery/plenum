# Interceptors

Request header injection, response tagging, authorization with short-circuit responses, and interceptor chaining with context passing.

## What it demonstrates

- `on_request_headers` hook to add headers to the upstream request
- `on_response` hook to add headers to the client response
- Short-circuit responses (returning 403 directly without hitting the backend)
- Chaining multiple interceptors on a single operation
- Context passing between interceptors via `ctx`
- `buffer-response: true` on the upstream (required for response interceptors)

## Architecture

```
Client -> Gateway (6188) -> WireMock backend
           |
           +-- /products: add-request-id (on_request_headers) -> tag-response (on_response)
           +-- /admin:    require-admin (on_request_headers) -> short-circuit or proxy
```

## Setup

```bash
docker compose up -d
```

## Try it out

### Chained interceptors on /products

```bash
curl -v http://localhost:6188/products 2>&1 | grep -i "x-request-id\|x-powered-by"
```

The response includes both:
- `x-request-id` — generated by `add-request-id.js`, passed through `ctx` and echoed by `tag-response.js`
- `x-powered-by: plenum` — added by `tag-response.js`

The `add-request-id` interceptor also forwards the request ID to the backend as a request header.

### Short-circuit on /admin (no key)

```bash
curl http://localhost:6188/admin
# {"error": "Admin access required"}
```

Returns 403 — the `require-admin` interceptor short-circuits the request before it reaches the backend.

### /admin with key

```bash
curl http://localhost:6188/admin -H "x-admin-key: anything"
```

The request passes through to the backend.
